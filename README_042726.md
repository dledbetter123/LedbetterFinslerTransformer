## The Real Problem: What Space Are We In?

The .md blueprint says:

> *"$a_{ij}(x)$: Learns the baseline semantic similarity between token clusters."*
> *"$b_i(x)$: The local vector field that biases flow toward the next logical token."*

This means $a_{ij}$ and $b_i$ should vary **in the token embedding space** — the manifold's coordinates ARE the semantic coordinates. But in our 2D experiments, the manifold's coordinates are just raw $(x_1, x_2)$ positions of spiral points. There's no semantic meaning to the positions.

### The consequence

The metric field learns to minimize geodesic deviation at observed trajectory points. Between and beyond those points, the metric is **unconstrained** — it can be anything. So the spray field outside the training data distribution is essentially random.

This is why the spray field looks like a smooth heatmap rather than sharp channels: the metric field is a smooth neural network with no reason to create structure outside the data support. The "channels" only exist where training trajectories are dense.

## Three Levels of the Problem

### Level 1: The spray field is correct but sparse

The analytic spray IS geometrically consistent. The issue is that we're evaluating it on a 100×100 grid where most points are far from any training trajectory. The metric field has never seen those positions, so its output there is just the smooth extrapolation of the neural network — which trends toward the initialization (near-identity metric, near-zero 1-form).

**Fix:** Visualize the spray only near trajectory support, or use a density-weighted visualization. The geometry is only meaningful where data exists.

### Level 2: The coordinate system matters

The .md says tokens should be embedded and then the embedding space becomes the manifold. In 2D, we skip the embedding — the raw coordinates ARE the manifold coordinates. This means:

- The metric field needs to learn the geometry of spirals in raw $(x_1, x_2)$ space
- Spiral curvature monotonically increases with radius
- The metric $a_{ij}(x)$ should stretch along the tangent direction and compress normal to it
- The 1-form $b_i(x)$ should point along the spiral's tangent

This is **possible** but it requires the metric field network to be expressive enough to capture the radially varying curvature. If the network is too smooth (too few parameters, too wide a receptive field), it can only learn a globally smooth metric that can't create sharp spiral channels.

**Fix:** Use a more expressive metric field — more layers, layer normalization, possibly a coordinate-conditioned architecture (like FiLM or hypernetwork conditioning on radius/angle).

### Level 3: The fundamental architecture question

Here's the deeper question from the notes: **should the spray even be computed from a learned metric?**

The .md blueprint actually says two things that are in tension:

1. *"Instead of querying an Attention matrix, our network will predict the Spray Coefficients $G^i$."* — suggesting a **learned spray** (the original MLP approach)
2. *"The spray is derived from the metric: $G^i = \frac{1}{4} g^{il}(\ldots)$"* — suggesting an **analytic spray from a learned metric** (our current approach)

These are DIFFERENT architectures with different trade-offs:

| | Learned Spray (MLP) | Analytic Spray (from metric) |
|---|---|---|
| Expressiveness | High — can learn any smooth vector field | Constrained — must be a geodesic spray |
| Geometric consistency | Not guaranteed | Guaranteed by construction |
| Capacity | Arbitrary functions | Only valid Finsler geometries |
| Training | Direct: spray → loss | Indirect: metric → spray → loss |
| Failure mode | Overfits to non-geometric patterns | Underfits if metric too smooth |

## Proposed Solution: Metric-Conditioned Spray

Instead of choosing between "pure MLP spray" and "pure analytic spray", use both:

```
G^i(x, y) = G^i_analytic(x, y; metric_field) + G^i_residual(x, y, a(x), b(x))
```

The **analytic spray** from the learned metric provides a geometrically grounded baseline — it guarantees Finsler structure, 2-homogeneity, and the correct geodesic equation in regions where the metric is well-learned.

The **residual spray** is a neural network that receives the local metric parameters `(a(x), b(x))` as additional conditioning. It can capture fine-grained, data-dependent corrections that the smooth metric field can't express.

**Why this works:**
- The analytic spray provides the "channels" from the metric topology
- The residual spray fills in the sharp local corrections
- The metric conditioning tells the residual network about local geometry, preventing it from ignoring the Finsler structure entirely
- During training, the metric and residual are jointly optimized
- The residual can be regularized toward zero to encourage the metric to do most of the work

## Alternative: Chart-Based Architecture

Another approach inspired by the notes: **multiple local coordinate charts**.

Instead of one global metric field, partition the embedding space into overlapping charts, each with its own local metric. This is closer to how differential geometry actually works — manifolds are covered by charts, not described by a single global coordinate system.

```python
class ChartBasedMetric(nn.Module):
    def __init__(self, d, n_charts=8, chart_hidden=64):
        self.chart_centers = nn.Parameter(torch.randn(n_charts, d))
        self.chart_metrics = nn.ModuleList([
            RandersMetricField(d, chart_hidden) for _ in range(n_charts)
        ])
    
    def forward(self, x):
        # Soft assignment to charts via distance weighting
        dists = torch.cdist(x, self.chart_centers)  # (B, n_charts)
        weights = torch.softmax(-dists / temperature, dim=-1)
        
        # Weighted sum of local metrics
        a_total = sum(w[:,:,None,None] * chart.forward(x)[0] 
                      for w, chart in zip(weights.T, self.chart_metrics))
        b_total = sum(w[:,:,None] * chart.forward(x)[1]
                      for w, chart in zip(weights.T, self.chart_metrics))
        return a_total, b_total
```

This gives the metric field **local specialization**: each chart can learn a different local geometry, and the overlap regions create smooth transitions. The number of charts determines the resolution of the geometry.

## Update 04:56 — Are Wind Walks Too Random? (Honest Analysis)

The spiral results show "significant improvements" — the metric field learns position-dependent geometry that creates channels along spiral paths. So the architecture works. The question is: **why does it fail on wind walks, and is that a fixable problem or a mathematical wall?**

### The Math: Why Spirals Work

A spiral is a **deterministic** curve. At any point $(x, \dot{x})$ on a spiral, the acceleration is a fixed, deterministic function of position and velocity. Multiple spirals share the same underlying dynamics (same curvature law, different phases).

The spray $G^i(x, y)$ is also deterministic — it's a function on the tangent bundle. So asking "find $G$ such that $\ddot{x} + 2G(x, \dot{x}) = 0$ along observed spirals" has a **well-defined, achievable solution**. The metric field can learn ONE geometry that explains ALL spirals simultaneously because they share the same dynamics.

Result: geodesic deviation → near zero. ✓

### The Math: Why Wind Walks Are Fundamentally Different

A wind walk is $x_{t+1} = x_t + w + \epsilon_t$ where $w = [0.05, 0.015]$ (drift) and $\epsilon_t \sim \mathcal{N}(0, 0.1)$ (noise).

The finite-difference velocity and acceleration are:
- $v_t = (x_{t+1} - x_t) / \Delta t = (w + \epsilon_t) / \Delta t$
- $a_t = (v_{t+1} - v_t) / \Delta t = (\epsilon_{t+1} - \epsilon_t) / \Delta t$

**The mean acceleration is exactly zero.** The drift $w$ is constant, so it cancels in the velocity difference. The acceleration is pure noise.

The geodesic deviation loss is:
$$\mathcal{L} = \| a_t + 2G(x_t, v_t) \|^2$$

The best any deterministic $G$ can do is match the **mean** acceleration, which is zero. Setting $G = 0$ is optimal. The loss converges to:
$$\mathcal{L}^* = \text{Var}(a_t) = \frac{2\sigma^2}{\Delta t^2} = \frac{2 \times 0.01}{0.01} = 2.0$$

**This is an irreducible floor, not a failure.** No spray — no matter how expressive — can beat it.

### Why This Isn't a Bug, It's a Feature Mismatch

The wind walks have three properties that make them a **bad** test case for position-dependent Finsler geometry:

1. **Constant drift.** The "wind" $w$ is the same everywhere. So the optimal 1-form $b_i$ is a global constant — no position dependence needed. The position-dependent metric field is overkill; a single global Randers metric suffices.

2. **No spatial structure.** At any position, the dynamics are identical (same drift, same noise distribution). There's nothing for the metric to learn about different regions of space. The "I've been here before" signal doesn't exist because all positions are equivalent.

3. **Low signal-to-noise ratio.** Drift magnitude $\|w\| = 0.052$. Noise std $\sigma = 0.1$. The noise is **twice the signal**. The geodesic can only capture the mean, and most of the trajectory is random walk.

Compare with spirals:
| Property | Spirals | Wind Walks |
|---|---|---|
| Dynamics | Deterministic (curvature) | Stochastic (noise) |
| Spatial variation | Strong (curvature ∝ radius) | None (uniform drift) |
| Signal-to-noise | Very high | ~0.5 |
| Optimal spray | Rich, position-dependent | Trivially $G \approx 0$ |
| Achievable loss floor | ~0 (just noise) | ~2.0 (irreducible) |

### Is the Intuition About "Routes as Sentences" Wrong?

**No — the intuition is exactly right, but the wind walks don't test it.** Here's the distinction:

**Language (what we're modeling):**
- "The cat sat on the ___" → there's a DETERMINISTIC tendency toward "mat" based on the context
- Different starting contexts lead to DIFFERENT continuations
- The metric should create different channels for different contexts
- "I've been here before" = recognizing a familiar context and following the learned channel

**Wind walks (what we're testing):**
- Every position has the SAME dynamics (uniform drift + noise)
- There's no context — no "sentence" structure to recognize
- The metric can only learn one global drift direction
- "I've been here before" has no meaning because all positions are equivalent

### What Would Actually Test the Theory?

The right stochastic test case needs **spatially varying structure** — the analog of different sentence contexts creating different dynamics in different regions of space:

**Option A: Position-dependent wind (Spatially varying Randers)**
```python
def generate_variable_wind_walks(n=64, T=30, noise=0.05):
    """Wind direction rotates with position — genuine spatial structure."""
    traj = torch.zeros(n, T, 2)
    traj[:, 0] = torch.randn(n, 2) * 0.5
    for t in range(T - 1):
        x = traj[:, t]
        angle = torch.atan2(x[:, 1], x[:, 0])  # Wind depends on position
        wind = 0.08 * torch.stack([torch.cos(angle + np.pi/4),
                                    torch.sin(angle + np.pi/4)], dim=-1)
        traj[:, t+1] = x + wind + noise * torch.randn(n, 2)
    return traj
```
Here the drift varies with position → the metric field HAS something to learn → different regions need different $b_i(x)$ → genuine position-dependent Finsler structure.

**Option B: Attractor dynamics (Ornstein-Uhlenbeck)**
```python
def generate_attractor_walks(n=64, T=30, noise=0.05):
    """Drift toward attractors — creates 'sentence-like' channels."""
    attractors = torch.tensor([[1., 1.], [-1., 1.], [0., -1.]])
    traj = torch.zeros(n, T, 2)
    traj[:, 0] = torch.randn(n, 2) * 0.3
    # Each walk is attracted to a random attractor
    targets = attractors[torch.randint(0, 3, (n,))]
    for t in range(T - 1):
        drift = 0.05 * (targets - traj[:, t])  # Pull toward attractor
        traj[:, t+1] = traj[:, t] + drift + noise * torch.randn(n, 2)
    return traj
```
Here different walks have different GOALS → the metric creates channels toward different attractors → "route" structure analogous to sentences.

### The Bottom Line

Your intuition is right: **in the right setup, training a Finsler manifold to find geodesics matching trajectories IS a mathematical certainty** — but only when the trajectories have learnable structure. The condition is:

$$\exists\ G(x,y) \text{ such that } \mathbb{E}[\ddot{x}_t \mid x_t, \dot{x}_t] = -2G(x_t, \dot{x}_t)$$

For spirals: the conditional expectation IS a deterministic function of $(x, \dot{x})$ → ✓ learnable
For constant-drift walks: the conditional expectation is zero everywhere → ✓ trivially learnable but uninteresting
For variable-drift walks: the conditional expectation varies with position → ✓ **this is the right test**

The wind walks aren't "too random" — they're **too uniform**. The randomness isn't the problem; the lack of spatial structure is.
